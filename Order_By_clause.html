Order By clause



Order By 可以依據多個順序做排序, 可選擇升冪(Asc)或者降冪(Desc)排序

Order By columnName1 [Asc|Desc], {columnName2 [Asc|Desc],...}

預設值為升冪

如果有 Where 子句, Order By 要放在 Where 子句後面

預設的輸出結果順序是區份大小寫

如果要輸出結果順序不區份大小寫, 可以使用 Lower 或 Upper 函數

Order By Lower ( columnName ) 

或 

Order By Upper ( columnName ) 

Order By 使用 Lower 或 Upper, 只會輸出順序, 不會改變輸出結果(大寫或小寫)

可以設定多個排序條件, 取得的值會依照由左至右的條件順序進行排序

越多排序條件, 越花時間

如果使用 Where 和 In 條件, 則取得的值的順序和 In 裡面條件值的順序相同

其他, 沒有指定排序條件, 則取得的值的順序為 rowId 的順序( 這是大部分的狀況)


計算式:

/* 
  Where 可以同食設定多個查詢條件, 用 And 或 Or 連接不同的查詢條件 
  Order By 子句放在 Where 子句的後面
  如果不寫 Desc 表示是Asc(升冪, 是預設值)
  可以設定多個排序順序, 之間以 "," 隔開
  Order By 後面寫的欄位名稱順序, 就是排序的先後順序 
*/
/*
  先以 nameEng 降冪排序, 再以中文名字的第一個字的筆畫數排序
*/
executeSql( "
    Select name, nameEng, gender, stroke, addrCity||addrStr, birthday 
    From contact 
    Where addrCity = ? And nameEng Like 'Arm%' 
    Order By nameEng Desc, stroke
    " ; ""; ""; "新北市" )

葉從,Armstrong,男,13,新北市金山區香果園158號9樓,1992-01-30
汪貢馨,Armand,男,7,新北市土城區順風路69號8樓,1990-02-04

可以用依據 lower 或 upper 來進行不分大小寫的排序
executeSql ( "SELECT webAddr FROM company ORDER BY UPPER (webAddr)" ; "" ; "" )

SELECT webAddr 
FROM company 
ORDER BY LOWER (webAddr) 
Fetch First 10 Rows Only

/*
可以用依據 lower 或 upper 來進行不分大小寫的排序
UPPER 先進行 UPPER 處理, 
DISTINCT 所以全部以大寫被列出, 所以原始資料是大小寫都會被合併
也可以用 LOWER 來處理
executeSql ( "SELECT webAddr FROM company ORDER BY UPPER (webAddr)" ; "" ; "" )
*/
executeSql("
SELECT DISTINCT UPPER ( webAddr ) 
FROM company 
ORDER BY UPPER (webAddr) 
Fetch First 10 Row Only
" ; "" ; "" )
